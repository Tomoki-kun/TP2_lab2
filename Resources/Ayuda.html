Casos de uso
Caso de uso: Reservar una propiedad Actor principal: Administrador
Proposito: Reservar una propiedad a un cliente
Poscondiciones: Queda reservada una propiedad y se registra un cliente
Resumen: El administrador registra un cliente y designa una propiedad que no este reservada
Curso normal:
Actor
1- El administrador busca una propiedad disponible según necesidades del cliente
3- El administrador selecciona una propiedad
4- El administrador completa los datos del cliente 		Sistema
2- Informa la lista de propiedades disponibles
5- Se guardan los datos del cliente y la reserva
Excepciones:
3- No se completan todos los datos
Curso Alternativo 1:
El usuario cancela el proceso en cualquier momento
Curso Alternativo 2:
No hay propiedades disponibles –Se cancela el proceso-

Caso de uso: Cancelar una Reserva Actor principal: Administrador
Propósito: Cancelar una reserva
Poscondiciones: Queda eliminada la reserva del sistema y la propiedad queda habilitada para recibir otras reservas
Resumen: El administrador busca un cliente y se elimina la reserva echa con el nombre de esa persona
Curso Normal:
Actor
1- El administrador ingresa los datos de la persona
2- El administrador selecciona la reserva a cancelar 		Sistema
3- Se cancela la reserva echa por el cliente
Excepciones:
1- No se completan todos los datos
2- No hay reservas registradas
Curso Alternativo 1:
El usuario cancela el proceso en cualquier momento

Caso de uso: Registrar una propiedad Actor principal: Administrador
Propósito: Registrar una propiedad
Poscondiciones: Queda registrada una propiedad y se registra un propietario si es casa
Resumen: El administrador registra una propiedad y un propietario si es casa
Curso Normal:
Actor
1- El administrador ingresa los datos de la propiedad
2 El administrador determina el tipo de propiedad 		Sistema
3- Verifica el tipo de propiedad.
4- Guarda los datos de la nueva propiedad
Excepciones:
3- No se completan todos los datos
Curso Alternativo 1:
El usuario cancela el proceso en cualquier momento
